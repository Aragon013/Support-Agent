REMOTE SUPPORT PRO - FRONT DOOR
Leer este archivo primero antes de continuar cualquier tarea del proyecto.

1) OBJETIVO
Construir un sistema profesional de soporte remoto tipo TeamViewer para MSP, con foco en seguridad, bajo costo operativo, compatibilidad y operacion en redes cerradas.

2) DECISIONES CERRADAS
- Arquitectura: P2P-first con relay fallback.
- Host Agent: servicio nativo en segundo plano.
- Controller: aplicacion de operador con Electron.
- Seguridad: cifrado E2E obligatorio + MFA.
- Logs: debug + auditoria con retencion configurable, default 90 dias.
- Licenciamiento: por endpoint.
- GitHub: sin releases hasta aprobacion explicita.

3) MODO DE TRABAJO
- Estrategia low budget de tokens.
- Usar MINOR para tareas pequenas.
- Usar MEDIUM para diseno, contratos API, pseudocodigo y refactors moderados.
- Usar MAX solo para arquitectura critica, seguridad o problemas complejos.
