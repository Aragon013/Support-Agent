AUTH + TENANT + ROLES - PLAN TECNICO V1
Fecha: 2026-07-03
Estado: Ready para implementacion

1) OBJETIVO
Implementar control de acceso para MSP multi-tenant con seguridad fuerte y bajo costo de operacion.

2) ALCANCE DEL MODULO
- Login con email/password
- MFA TOTP para operadores
- Sesiones JWT + refresh token
- Multi-tenant isolation
- Roles base: Admin, Tech, Viewer
- Politicas por tenant (MFA requerido, caducidad sesion, bloqueo intentos)

3) REGLAS FUNCIONALES
3.1 Login
- Input: email, password
- Si credenciales validas y MFA habilitado: challenge TOTP
- Si credenciales validas y MFA no habilitado: emite JWT + refresh
- Si falla: incrementa contador de intentos

3.2 Bloqueo por seguridad
- 5 intentos fallidos en 15 min => lock temporal 15 min
- Eventos de lock quedan en audit log

3.3 Refresh
- Access token: 15 min
- Refresh token: 30 dias (rotativo)
- Reuso de refresh invalido => revocar cadena de sesion

3.4 Tenant boundaries
- Todo operador pertenece a tenant_id
- Toda consulta y mutacion filtra por tenant_id
- Nunca retornar recursos de otro tenant

3.5 Roles
- Admin: gestiona operadores, politicas, endpoints, auditoria
- Tech: controla endpoints, comandos, transferencias
- Viewer: solo lectura de estado/auditoria

4) REQUISITOS NO FUNCIONALES
- p95 login < 400 ms
- Hash de password: Argon2id
- JWT firmado con llave rotativa (kid)
- Auditoria de eventos criticos en tabla dedicada

8) PSEUDOCODIGO CLAVE
8.1 login(email, password)
- user = findOperatorByEmailAndTenant(email, tenant)
- if user null: audit failed; return 401
- if now < user.locked_until: audit lock active; return 423
- if !verifyArgon2(password, user.password_hash):
  - increment failed_attempts
  - if failed_attempts >= policy.max_failed_attempts: locked_until = now + lock_minutes
  - audit failed
  - return 401
- reset failed_attempts
- if policy.require_mfa OR user.mfa_enabled:
  - issue short-lived mfa_challenge_token (5 min)
  - audit mfa challenge
  - return 202 mfa_required
- tokens = issueAccessAndRefresh(user)
- audit login success
- return 200 tokens

8.2 refresh(refresh_token)
- h = sha256(refresh_token)
- session = findSessionByHash(h)
- if session null OR revoked OR expired: audit invalid; return 401
- if token already used (rotation check):
  - revoke family/session chain
  - audit refresh reuse detected
  - return 401
- revoke old refresh
- issue new access + refresh
- save new session row
- audit refresh success
- return 200 tokens
