Convenciones
- Prioridad: P0 (bloqueante), P1 (alta), P2 (media), P3 (baja).
- Esfuerzo estimado: S (0.5-1 dia), M (1-2 dias), L (3-5 dias).
- DoD = Definition of Done del ticket.
- Restricciones globales aplican a todos los tickets: E2E, MFA step-up, logs 90d, licencia endpoint, no-release.

SLICE B - Policy + MFA gate
Objetivo: bloquear ejecuciones no autorizadas y elevar autenticacion cuando aplique.

B-01 | Policy evaluator Command Engine [BASELINE FUNCIONAL LISTO]
- Prioridad: P0
- Esfuerzo: L
- Dependencias: A-01, A-02
- Alcance:
  - Evaluar allow_remote_command, allowed/blocked command ids.
  - Evaluar role threshold por risk.
  - Evaluar max_run_seconds, max_output_bytes, concurrencia endpoint.
- DoD:
  - Cobertura de tests de policy >= 90% ramas criticas.
  - Resultado explicito: allow | deny | stepup.

B-02 | Schema validator de parametros [BASELINE FUNCIONAL LISTO]
- Prioridad: P0
- Esfuerzo: M
- Dependencias: A-03
- Alcance:
  - Validar tipos, rangos, regex, enums por command_id versionado.
  - Rechazar parametros extra o ambiguos.
- DoD:
  - 100% de comandos catalogados tienen schema.
  - Error 422 detallado por campo.

B-03 | Integracion MFA step-up [BASELINE FUNCIONAL LISTO]
- Prioridad: P0
- Esfuerzo: M
- Dependencias: B-01
- Alcance:
  - Si risk requiere step-up, marcar mfa_pending y lanzar challenge.
  - Reusar challenge solo dentro de ventana corta configurada.
- DoD:
  - Audit trail completo: required/passed/failed.
  - No se puede dispatch sin mfa_verified_at valido.

B-04 | Gates por licencia endpoint [BASELINE FUNCIONAL LISTO]
- Prioridad: P0
- Esfuerzo: S
- Dependencias: A-03

- DoD:
  - Respuesta 403 con motivo license_inactive.
  - Evento de auditoria command.policy.denied por licencia.

SLICE C - Host execution pipeline
Objetivo: ejecutar comandos de forma segura, controlada y observable.

C-01 | CommandDispatcher en host-agent
- Prioridad: P0
- Esfuerzo: M
- Dependencias: A-04, B-01, B-03
- Alcance:
  - Recepcion de command.init cifrado.
  - Reserva de slot segun concurrencia.
  - Transicion dispatched -> running.
- DoD:
  - Cola endpoint estable en carga moderada.
  - Sin perdida de jobs en reconexion breve.

C-02 | CommandRunner por catalogo
- Prioridad: P0
- Esfuerzo: L
- Dependencias: C-01, B-02
- Alcance:
  - Ejecutar handlers dedicados por command_id.
  - Prohibir shell libre interactiva en V1.
  - Aplicar least privilege y entorno controlado.
- DoD:
  - Demo de 3 comandos (diagnostic, maintenance, security-safe).
  - Intentos de injection bloqueados por diseno.

C-03 | OutputStreamer con caps
- Prioridad: P1
- Esfuerzo: M
- Dependencias: C-02
- Alcance:
  - Stream stdout/stderr por chunks secuenciales.
  - Aplicar max_output_bytes y truncado controlado.
  - Calcular digest sha256 del output total.
- DoD:
  - UI recibe stream incremental y exit final.
  - Si supera cap: marca truncado sin romper job.

C-04 | Cancelacion, timeout y watchdog
- Prioridad: P0
- Esfuerzo: M
- Dependencias: C-02
- Alcance:
  - Cancelacion operador: graceful stop + kill forzado.
  - Timeout runtime por policy.
  - Limpieza de procesos huerfanos y temporales.
- DoD:
  - Tests de race cancel/finish estables.
  - Estado final coherente (cancelled o failed_timeout).

RIESGOS DE EJECUCION DEL BACKLOG
- Scope creep en comandos custom del tenant.
  - Control: congelar catalogo V1 y versionar extensiones.
- Sobrecarga por output excesivo.
  - Control: caps estrictos + truncado + digest.
- Acoplamiento fuerte control-plane/host.
  - Control: contratos versionados y adaptadores estables.
