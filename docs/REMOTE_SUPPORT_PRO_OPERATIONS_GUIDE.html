RemoteSupportPro

Guia Total de Operacion y Uso

Fecha: 2026-07-08 | Version: 1.0

1) Que es este sistema

RemoteSupportPro es una plataforma de soporte remoto para MSP con foco en seguridad operativa, trazabilidad y control por perfil de instalacion.

Host Agent
Servicio nativo en segundo plano por endpoint.
Control Plane
API central de sesiones, comandos, auditoria y politicas.
Controller Electron
Cliente de operador con soporte + cybersecurity.
Objetivo
Resolver soporte remoto sin perder control de riesgo.

2) Para que sirve

  • Soporte remoto diario (view/control).
  • Comandos operativos con gating por perfil y tenant.
  • Auditoria de acciones y seguridad.
  • Compliance, excepciones, alertas y resiliencia defensiva.

3) Modelo de liberacion acordado

Windows

  • 2 programas separados: Host Service + Controller Client.
  • El instalador del host permite elegir perfil: remote_only, support_limited_no_folders, support_full.
  • Ese perfil controla que funciones estaran habilitadas remotamente.

macOS

  • Un instalador por perfil de host (paquetes separados).
  • Simplifica permisos/plist/policies por tipo de despliegue.

Linux

  • Host por paquete/config equivalente con perfil definido.

4) Seguridad base

  • E2E obligatorio.
  • MFA obligatorio.
  • Logs y auditoria con retencion configurable (default 90 dias).
  • Aislamiento multi-tenant.
  • Gating por installProfile + rol + policy.

5) Secciones del Controller y como usarlas

Support

Entrada rapida para soporte operativo. Crea sesion y dispara acciones iniciales.

Commands

Catalogo de comandos con parametros y riesgo. Ejecuta solo lo minimo necesario.

Jobs

Seguimiento en vivo por WebSocket del estado de ejecucion.

Cybersecurity (seccion unificada)

Toda la auditoria cyber en un solo lugar, con subtabs:

  • SecAudit: crear/ejecutar planes y comparar baseline.
  • Compliance: evaluacion contra CIS, NIST CSF, ISO 27001, SOC2, PCI DSS.
  • Exceptions: aprobar/rechazar excepciones temporales.
  • Alerts: canales y envio de alertas de drift.
  • Resilience: ejercicios defensivos dry-run.
  • Audit Log: evidencia cronologica de eventos.
Flujo sugerido: SecAudit -> Compliance -> Exceptions -> Alerts -> Audit Log

Monitor

Observabilidad operativa de estado y salud.

Sessions

Control avanzado por capacidades de sesion y perfil del endpoint.

Settings

Configuracion del cliente operador.

6) Host profiles y capacidades

  • remote_only: acceso remoto basico, sin acciones de alto impacto.
  • support_limited_no_folders: soporte tecnico limitado, sin operaciones de carpeta.
  • support_full: soporte completo segun rol/politica.

7) Runbook de operacion recomendado

  1. Validar endpoint y perfil.
  2. Abrir sesion segura.
  3. Ejecutar comandos minimos necesarios.
  4. Registrar evidencia en auditoria.
  5. Correr SecAudit ante incidente o hardening.
  6. Revisar compliance y excepciones.
  7. Configurar alertas de drift.

8) Diagnostico rapido

  • UI negra: revisar ErrorBoundary y modulos del panel activo.
  • 401 en alerts/resilience: validar API key y tenant.
  • Comando bloqueado: revisar installProfile + policy gates.
  • Sin evidencia: validar ingest de audit events en backend.
